# UX 카피 / 고정 문구 모음

`콘텐츠작성_투두리스트.md` 7번 항목의 실제 문구. 결과 화면 공유 유도, 안내(디스클레이머), 비속어 필터 블록리스트, 단계별 CTA 문구를 모두 포함. 톤은 기존과 동일(친근한 존댓말, 단정적·공포 조성 표현 금지, 트렌디 표현은 가볍게만).

## 1. 결과 화면 공유 유도 문구

기능별로 살짝 다르게 변주해 같은 문구가 반복되는 느낌을 줄인다. 모두 "강제"가 아니라 제안형 — 버튼 옆 보조 텍스트로 사용.

| 기능 | 공유 유도 문구 |
|---|---|
| 오늘의 운세 (메인) | 이 결과 캡처해서 친구한테 보내보기 |
| 타로 3카드 | 내가 뽑은 카드, 친구는 뭐 나올지 궁금하지 않아? |
| 행운 넘버 / 운명 비틀기 | 오늘 숫자 캡처해서 잘 챙겨두기 |
| 오늘의 랭킹 | 내 순위 친구한테 자랑하기 |
| 궁합 모드 (전체) | 이 결과 캡처해서 같이 본 사람한테 보내보기 |
| 베프궁합 | 이 케미 결과, 그 친구한테 바로 보내보기 |
| 짝사랑궁합 | (공유 유도 생략 — 민감한 주제라 별도 안내문구만 노출, 4번 참고) |
| 타로 성향 테스트 | 이 결과 캡처해서 친구한테 보내보기 |
| 78장 갤러리 (카드 상세) | 이 카드 의미 친구한테도 보여주기 |

## 2. 안내(디스클레이머) 문구 확정안

### 2.1 공통 기본 문구 (모든 결과 화면 고정 노출)
> 참고용 콘텐츠이며 실제 결정의 근거로 사용하지 마세요.

### 2.2 행운 넘버 / 운명 비틀기 전용 — 도박 연관 표현 회피
- 결과 화면 고정 문구: 위 공통 문구만 노출, "로또"·번호 출처·구매 유도 문구는 절대 추가하지 않음(`기획안.md` 5번 정책).
- 금전운 점수 미달(70점 미만)로 버튼이 비활성일 때 안내 문구:
  > 아직 행운 넘버를 볼 수 있는 흐름은 아니에요. 대신 오늘의 카드로 위로의 숫자를 받아보세요.
  (→ "운명 비틀기"로 자연스럽게 유도, `기획안.md` 4.5 연계)

### 2.3 짝사랑궁합 전용 (이미 확정, 재게재)
> 이 결과는 참고용이며, 실제 상대방의 마음과는 다를 수 있어요. 관계는 결과보다 직접 나누는 대화로 만들어가는 게 가장 정확해요.
(`궁합_관계유형별_해석가이드.md` 4.5와 동일 — 공통 기본 문구에 추가로 노출)

### 2.4 오늘의 랭킹 / 궁합 랭킹 — 등록 실패 안내
- 비속어 필터 차단 시: 다른 이니셜을 입력해주세요.
- 일일 등록 횟수(IP당 제한) 초과 시: 오늘은 더 이상 랭킹에 등록할 수 없어요. 내일 다시 도전해보세요.
- 순위 밖으로 밀려난 경우(결과는 봤지만 등록 불가): 오늘의 TOP 5에는 들지 못했지만, 결과는 그대로 저장해서 간직해보세요.

## 3. 비속어 필터 블록리스트 (영문 3자 이니셜)

`오늘의랭킹_기획.md` 4번 / `관계궁합_확장_기획.md` 2번에서 정의된 영문 3자 이니셜(A~Z만 허용) 입력에 적용. 대소문자 구분 없이 검사하고, 아래 리트스피크 치환 규칙을 먼저 적용한 뒤 블록리스트와 대조한다.

### 3.1 치환 규칙 (필터 적용 전 정규화)
- 이니셜은 영문자만 허용되므로 숫자 치환 자체는 발생하지 않지만, 시각적으로 비슷한 알파벳 조합(예: "VV" → "W"처럼 보이는 경우)은 운영 중 사용자 신고로 추가 보강.
- 검사 시 대문자로 통일(예: "ass" → "ASS")해서 대조.

### 3.2 차단 목록 (직접 비속어/혐오 표현 약어)

```
ASS, FUK, FUC, FCK, SHT, SHI, BIT, CUM, SEX, GAY, FAG, NIG, KKK,
RAP, PED, SOB, DIE, KYS, HOE, SLT, DIK, DCK, PUS, TIT, BBC, NWO,
ANL, ORL, JIZ, WHR, SLU, PMP, GFY, STFU, WTF, FFS, OMG(X),
NAZ, HTL, ISI, ALQ
```

(주: WTF/STFU/GFY/OMG처럼 4자 약어는 3자 입력칸에는 들어가지 않지만, 부분 일치 검사 로직 도입 시를 대비해 참고용으로 함께 기록. 실제 3자 비교 시에는 4자 항목 제외하고 운영.)

### 3.3 운영 원칙
- 위 리스트는 1차 출시용 최소 세트. 실제 운영 중 신고/모니터링으로 계속 추가하는 **운영 단계 지속 관리 항목**으로 명시.
- 정치적으로 민감한 실존 인물 이니셜(예: 전/현직 대통령 이니셜 조합)도 운영 중 추가 검토 필요 — 1차 리스트에는 포함하지 않되 모니터링 대상으로 기록.
- 한글 닉네임은 입력 자체가 불가(영문 3자만 허용)하므로 한글 비속어 필터는 불필요.

## 4. CTA 문구 전체 세트 (결과 화면 연쇄 유도, 4.8)

`체류시간_증대_기획.md` 2번 표를 실제 버튼/문구로 구체화.

| 결과 화면 | CTA 버튼 문구 | 건너뛰기 문구 |
|---|---|---|
| 오늘의 운세 점수 | 타로 3카드도 뽑아보기 | 다음에 보기 |
| 타로 3카드 결과 (금전 70점 이상) | 행운 넘버 보러가기 | 괜찮아요, 여기까지 볼게요 |
| 타로 3카드 결과 (금전 70점 미달) | 오늘의 카드로 운명 비틀기 | 괜찮아요, 여기까지 볼게요 |
| 행운 넘버 / 운명 비틀기 결과 | 오늘의 TOP 5 랭킹 확인하기 | 랭킹은 다음에 볼게요 |
| 오늘의 랭킹 | 애인/친구와 궁합 모드 해보기 | 궁합은 다음에 볼게요 |
| 궁합 모드 결과 | 78장 카드 갤러리 둘러보기 | 갤러리는 다음에 볼게요 |
| 궁합 모드 결과 (관계유형 전환) | 다른 관계로도 보기 (예: 연인 → 베프) | — (선택 사항이라 건너뛰기 불필요) |
| 78장 갤러리 / 카드 상세 | 내 별자리 운세도 보기 | — (탐색형 콘텐츠라 종착점, 건너뛰기 불필요) |
| 타로 성향 테스트 결과 | 78장 카드 갤러리 둘러보기 | 갤러리는 다음에 볼게요 |

- 모든 "건너뛰기" 문구는 부담 없는 톤 유지("아니요", "취소" 같은 단정적 단어 대신 "다음에 볼게요" 류로 통일).
- "다른 관계로도 보기" 버튼 클릭 시 관계 유형 선택 화면으로 돌아가되, 이미 입력한 두 사람 생년월일은 유지(재입력 불필요) — 코딩 단계 UX 메모.
