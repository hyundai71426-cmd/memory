# 행운 넘버 매핑 규칙

기획안.md 4.4(행운 넘버 생성)의 세부 규칙. 기존 "6개 중 2~3개"라는 모호한 표현을 **"의미 있는 번호 3개 + 결정론적 랜덤 3개"**로 고정해 구현 시 헷갈리지 않게 한다.

## 0. 노출 조건 (변경, 2026-06-17)

게이트 없음. 누구에게나 항상 6개를 공개한다. 최종 금전운 점수 ≥ 70점이면 결과 화면에 특별 표시(골드 효과/뱃지 등 시각적 강조만, 숫자 구성 로직에는 영향 없음)를 추가한다. (기획안 4.4)

기존 "70점 미달 시 버튼 비활성화" 방식은 폐기 — 하루 1회 고정(기획안 4.3) 정책과 결합되면 재도전 없이 그날 막혀버리는 막다른 길 경험이 되기 때문.

## 1. 의미 있는 번호 3개 — 타로 카드 → 숫자 변환

뽑힌 과거/현재/미래 카드 3장을 각각 1~45 사이 숫자로 변환한다. 정/역방향은 숫자에 영향 주지 않음(운세 점수에만 영향, 번호는 카드 자체 정체성에서 추출).

| 카드 종류 | 숫자 변환 규칙 |
|---|---|
| 메이저 아르카나 0 (The Fool) | 45 (0은 단독 숫자로 쓸 수 없어 특별히 45로 매핑) |
| 메이저 아르카나 I~XXI | 카드 번호 그대로 (1~21) |
| 마이너 숫자카드 (Ace~10) | 숫자 그대로 (Ace=1 ~ 10=10) |
| Page | 11 |
| Knight | 12 |
| Queen | 13 |
| King | 14 |

→ 과거/현재/미래 카드 각각 위 표로 변환 → 숫자 3개 후보 생성.

### 중복 처리

3장 중 2장 이상이 같은 숫자로 변환되는 경우(예: 검 Knight와 펜타클 Knight가 둘 다 12), 다음 순서로 대체 숫자를 보충한다.

1. 생년월일의 "일"(1~31)
2. 생년월일의 "월"(1~12)
3. 생년월일 연도 끝 두 자리를 45로 나눈 나머지 + 1

이미 후보 목록에 있는 숫자는 건너뛰고 다음 후순위로 넘어간다.

## 2. 결정론적 랜덤 번호 3개

- 시드: 생년월일 + 오늘 날짜 (기획안 4.2의 베이스 점수 시드와 동일한 방식·동일 시드 사용 가능 → 같은 사람·같은 날엔 항상 같은 결과, 4.3의 "하루 1회 고정"과 자연히 호환).
- 시드 기반 PRNG로 1~45 범위에서 추출하되, **1번에서 만든 의미 있는 번호 3개와 중복되지 않도록** 건너뛰며 추출.

## 3. 최종 처리

1. 의미 있는 번호 3개 + 랜덤 번호 3개 = 총 6개 (중복 없음, 모두 1~45).
2. 오름차순 정렬해서 화면에 표시.
3. **출처 설명 노출 금지**: 결과 화면에는 최종 숫자 6개만 보여주고, 어떤 카드/생년월일에서 왔는지는 설명하지 않음(힌트 없음 원칙 — 애드센스 도박 콘텐츠 분류 회피 + 신비감 유지 목적). 산출 로직 자체는 내부적으로 1~2번 규칙 그대로 유지하되, 화면에는 절대 노출하지 않음.

## 4. 기획안 4.4 갱신 필요 사항

기존 문구 "6개 중 2~3개는 타로 카드 번호나 생년월일 숫자에서 추출, 나머지는 랜덤"을 위 1~3번 규칙으로 명확화함. "로또"라는 단어는 기획안 전체에서 사용하지 않으며, 이 기능명은 "행운 넘버"로 통일.
