# 오늘의 카드 · 궁합 모드 기획

기획안.md 4번(핵심 로직)에 추가되는 두 기능의 세부 설계.

## 1. 오늘의 카드 (역할 재정의, 2026-06-17)

### 1.1 컨셉

생년월일 입력 직후, 본 운세·타로 드릴다운(4.1)을 시작하기 **전에** 가볍게 카드 1장을 먼저 공개한다. 운세 점수 계산에는 전혀 영향을 주지 않는 순수 플레이버 요소이자, 메인 플로우 진입 전 분위기를 잡는 워밍업 인터랙션.

**(폐기됨)** 기존에는 행운 넘버 게이트(금전운 70점) 미달자를 위한 위로 보상 장치로도 쓰였으나, `기획안.md` 4.4에서 게이트 자체를 없애 위로 보상이라는 용도가 사라짐. "오늘의 카드"는 이제 그 용도와 무관한 독립 기능이다.

### 1.2 흐름

1. 생년월일 입력 후 "오늘의 카드 보기" 클릭 → 78장 중 1장이 즉시 뒤집힘(드릴다운 없이 클릭 1회로 공개). 정/역방향은 50% 랜덤.
2. 이후 사용자는 평소대로 본 운세 + 3카드 드릴다운(과거/현재/미래)을 진행 → 모두 끝난 뒤 최종 점수(베이스+타로) + 행운 컬러 + 행운 넘버를 한 화면에서 한 번에 확인(`기획안.md` 3.1, 4.2 참고).

### 1.3 캐시/고정 규칙

- 오늘의 카드는 생년월일+오늘 날짜 키로 로컬 저장 → 하루 1회 고정(기획안 4.3과 동일한 정책), 같은 날 재방문 시 동일하게 표시.
- 점수 계산(4.2)에는 포함하지 않음 — 순수 플레이버 요소로 분리해 기존 점수 로직을 건드리지 않음.

## 2. 궁합 모드

### 2.1 컨셉

두 사람의 생년월일을 입력해 "오늘 두 사람의 궁합"을 보는 기능. 영원히 고정된 궁합 점수가 아니라 **매일 바뀌는 점수**로 설계한다 — 연인 사이에 한 번 나쁜 점수가 박제되면 오히려 부담스럽고, 매일 바뀌는 쪽이 가볍게 재방문할 이유도 만들어준다.

### 2.2 점수 산출

- **시드**: 두 사람의 생년월일(정렬해서 입력 순서 무관하게 처리) + 오늘 날짜.
- **베이스 궁합 점수**: 위 시드 기반 결정론적 랜덤, 0~100점.
- **케미 카드**: 같은 시드로 78장 중 1장을 뽑음(클릭 1회 공개, 드릴다운 없음). 이 카드의 애정 가중치(`타로카드_가중치.md`의 애정 컬럼)를 베이스 점수에 더해 최종 궁합 점수 산출 후 0~100 클램핑.
- **해석 텍스트**: "오늘 두 사람의 케미는…" 식의 짧은 해설. 생성 방식은 `운세텍스트_생성방식.md`의 AI+템플릿 하이브리드를 그대로 재사용.

### 2.3 캐시/고정 규칙

- 두 생년월일 조합(정렬) + 오늘 날짜를 키로 로컬 저장 → **하루 1회 고정**, 같은 날 재조회 시 동일 결과(기획안 4.3과 동일 정책).
- 행운 넘버 기능과는 무관한 독립 기능(궁합 결과가 행운 넘버에 영향 주지 않음).

### 2.4 페이지/배치

- 메인 페이지의 별개 탭/섹션 또는 `/compatibility` 독립 페이지로 제공 (세부 UI는 추후 결정).
- 결과 카드 저장/공유 기능(행운넘버_매핑규칙.md 외 별도 결과 이미지) 적용 대상이며, 향후 확장 1순위의 QR코드도 이 결과 이미지에 함께 적용 가능.

## 3. 기획안 반영 필요 사항

- 사이트맵에 `/compatibility` (또는 메인 내 탭) 추가.
