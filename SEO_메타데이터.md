# SEO 메타데이터

`콘텐츠작성_투두리스트.md` 9번 항목. 메인/별자리12/띠12/타로카드78/궁합 페이지의 title·description, 그리고 페이지별 OG(공유 미리보기) 이미지 디자인 사양.

- title은 한글 기준 25~32자, description은 70~100자 내외를 기준으로 작성(검색결과 노출 잘림 방지).
- 모든 description 끝에는 가능한 한 행동유도(예: "확인해보세요", "보러가기")를 넣어 클릭률을 높인다.
- 사이트명은 **하루조각**으로 확정되어 본문에 반영했습니다(`정책_안내_페이지.md`와 동일).

## 4. 타로 카드 78장 (`/tarot/[card-name]`)

78개를 일일이 손으로 다르게 쓰면 분량이 너무 커지고 관리도 어려워서, **템플릿 + 카드별 키워드 치환** 방식으로 설계. 키워드만 카드마다 다르게 넣어도 검색엔진이 보기엔 충분히 구별되는 고유한 문구가 된다(완전 동일 문구 반복은 피함).

### 4.1 공통 템플릿

- description: `{카드명} 타로카드의 정방향·역방향 의미({키워드})와 연애·금전·직장 운세 해석을 확인해보세요.`

## 6. OG 이미지(공유 미리보기) 디자인 사양

기존에 정의된 9:16/정사각형 **결과 카드**(사용자가 자기 결과를 캡처해서 공유하는 이미지)와는 별개로, 카카오톡/링크 공유 시 자동으로 뜨는 **페이지 자체의 OG 미리보기 이미지**가 필요함.

### 6.1 공통 규격

- 권장 사이즈: 1200×630px (표준 OG 비율, 페이스북/카카오톡/슬랙 등에서 잘림 없이 표시됨)
- 포맷: PNG 또는 JPG, 용량 300KB 이하 권장(로딩 속도)
- 공통 요소: 사이트 로고/워터마크(우측 하단 소형), 미스틱 인디고 톤 배경(`기획안.md` 7번 — 배경 #211b3d, 골드 포인트 #c9a227, 보조 인디고 #4a3f7a)

### 6.2 페이지 유형별 구성

| 페이지 유형 | 이미지에 들어갈 핵심 텍스트/요소 |
|---|---|
| 메인 페이지 | 사이트 로고/이름 + "오늘의 운세 무료로 보기" + 타로 카드 일러스트(은둔자 or 별 등 신비로운 카드 1~2장) |
| 별자리 페이지 (12종) | 해당 별자리 기호/일러스트 + "{별자리명} 오늘의 운세" 텍스트 — 12개 각각 별자리 심볼만 교체하는 템플릿 1개로 제작 |
| 띠 페이지 (12종) | 해당 동물 일러스트(쥐, 소, 호랑이 등 심볼화) + "{띠명} 오늘의 운세" 텍스트 — 템플릿 1개 + 동물 아이콘 교체 |
| 타로 카드 페이지 (78종) | 카드 실제 이미지(RWS 1909 기반 카드 아트) + 카드명 + "타로카드 의미" 텍스트 — 템플릿 1개 + 카드 이미지/이름만 교체(78개 각각 손으로 디자인할 필요 없이 자동 생성 스크립트로 처리 권장, 단 실제 생성은 코딩 단계 작업) |
| 궁합 페이지 (4유형) | 유형별 아이콘(하트=연인, 손하트=베프, 서류=상사, 반쪽하트=짝사랑) + "{유형명} 궁합 보기" 텍스트 — 템플릿 1개 + 아이콘/텍스트 교체 |

### 6.3 비고

- 실제 이미지 자동 생성(템플릿에 텍스트/이미지 합성)은 코딩 단계 작업이라 본 문서에서는 디자인 사양만 정리. 코딩 착수 승인 후 구현.
- 78장 카드 OG 이미지는 카드 원본 이미지를 그대로 활용 가능해 추가 디자인 리소스가 거의 들지 않음 — 우선순위 낮게 잡아도 무방.

### 6.4 v1 포함 범위 확정 (2026-06-17)

`출시_분할_로드맵.md` v1 Build 목록에 OG 이미지가 별도 항목으로 빠져 있던 것을 확인 후 보강함. v1 SEO 콘텐츠 페이지(타로/별자리/띠)가 이미 v1 범위이고, 바이럴 검증이 v1 핵심 목표이므로 og:image 부재 시 카카오톡/SNS 공유 시 미리보기가 깨져 검증 자체가 무의미해짐.

- **메인 / 별자리 12종 / 띠 12종 / 타로 78종 OG 이미지 → v1 포함** (`출시_분할_로드맵.md` v1 Build 항목에 추가 필요)
