# 출시 분할 로드맵 (v1 → v2 → v3)

> 기존 `기획안.md` 8번의 "1~2순위 전부 완성 후 빅뱅 출시" 전략을 대체하는 분할 출시안.
> 핵심 원칙: **검증되지 않은 3대 가정(애드센스 승인 / 바이럴 / 단일 채널 수익성)을 한꺼번에 베팅하지 않는다.** 각 단계는 다음 단계로 넘어가기 위한 **게이트 지표**를 통과해야 한다. 게이트 미달 시 다음 단계 착수 대신 해당 단계를 고친다.

---

## 0. 왜 빅뱅을 버리는가 (의사결정 근거)

| 빅뱅의 가정 | 실제 리스크 | 분할이 주는 것 |
|---|---|---|
| 애드센스는 당연히 승인된다 | 운세+자동생성 텍스트+1~45 숫자6개 = 정책 회색지대. 거부 시 매출 0 | v1에서 가장 싸게 단일 실패점부터 제거 |
| SNS 공유가 잘 될 것이다 | 미검증. 타겟(2030여성)과 로또 후크의 정합성도 불확실 | v1 정적+공유이미지만으로 바이럴 가정 실측 |
| 애드센스 단독으로 수익이 난다 | 한국 운세 RPM 낮음. AI 비용이 광고 수익 추월 가능 | v1 실트래픽으로 유닛 이코노믹스 확정 후 v2 투자 |
| 완성도로 선점한다 | 한 주면 복제 가능 → "완성도 선점"은 해자가 아님 | 빨리 검증하고 빨리 고치는 게 선점보다 안전 |

빅뱅은 수개월치 개발비를 첫 시장 신호 전에 모두 태운다. 분할은 가장 비싼 모듈(서버DB·4종 궁합·4개 리더보드·랭킹·퀴즈)을 검증 신호 뒤로 미뤄 매몰비용을 줄인다.

## 1. v1 — "승인 + 바이럴 검증용 얇은 코어" (출시 1차)

목표: ①애드센스 승인을 받는다 ②바이럴 가정을 실측한다 ③유닛 이코노믹스 데이터를 모은다.
서버 DB 없이, 전부 정적/로컬/시드 계산만으로 동작 → 인프라·운영 부담 최소.

### 포함 (Build)
- 메인 인터랙티브: 생년월일 입력 → 오늘의 운세(종합 + 4카테고리 점수·해설)
- 타로 3카드 스프레드 — **단, 드릴다운은 1~2클릭으로 단순화**(2덱→13×3 4클릭 구조는 v1에서 제외, 아래 "단순화" 참고)
- 행운 넘버(금전운 70점 게이트 + 운명비틀기 위로 1개) — **단, 애드센스 사전 승인 결과에 따라 조건부**(아래 ★ 참고)
- 결과 카드 저장/공유 이미지(9:16 기본 + 정사각형, UTM 워터마크)
- **OG 공유 미리보기 이미지**(메인 1 + 별자리 12 + 띠 12 + 타로 78, 카카오톡/SNS 링크 공유 시 노출. 규격·구성은 `SEO_메타데이터.md` 6번 참고) — 바이럴 검증이 v1 핵심 목표라 누락 시 검증 자체가 무의미해짐. 궁합용 OG는 궁합 기능과 함께 v2/v3로 연기.
- SEO 콘텐츠 페이지: 타로 78장 의미 + 별자리 12 + 띠 12 + 가이드 2 (이미 초안 완료분 활용)
- 78장 갤러리(`/tarot/`) — 정적 콘텐츠라 DB 불필요, 체류시간 핵심
- 정책 페이지 4종(privacy/terms/about/contact)
- GA4 + 핵심 이벤트(저장 클릭, 공유유도 클릭)
- AI 텍스트 생성 + 폴백 템플릿 + 비용 상한 자동전환 (이미 설계 완료)

### 제외 → v2 이후로 (Cut)
- 서버 DB 전체
- 오늘의 랭킹 TOP5
- 궁합 모드(4종) + 관계유형별 4개 리더보드
- 타로 성향 테스트 퀴즈
- 결과 화면 연쇄 유도(퍼널)의 *랭킹·궁합으로 가는 CTA* (운세→타로→갤러리 흐름만 v1 유지)

### ★ 애드센스 사전 검증 (v1 착수 전 선행 과제)
1. **콘텐츠 페이지만 먼저 배포해 애드센스 심사 신청** — 인터랙티브 없이 별자리/띠/타로/가이드 정적 페이지로 승인 도전.
2. 승인 후 **행운 넘버 기능 추가 → 정책 위반 알림 여부 모니터링.** 알림 시 즉시 기능 격리/제거하고 운세+타로만으로 운영.
3. 이 절차로 "전 매출이 걸린 단일 실패점"을 수개월 개발 전에 가장 싸게 제거한다.

### 드릴다운 단순화 (v1)
- 2덱 선택(39장)은 결과가 시드로 고정돼 "가짜 선택권"이므로 v1에서 제거.
- 과거/현재/미래 카드 3장만 한 화면에서 각 1클릭(총 3클릭) 또는 "한 번에 펼치기" 1클릭 옵션 제공.
- 풀 드릴다운 의례(ritual)는 효과 검증 후 v2에서 부활 여부 결정.

### v1 → v2 게이트 지표 (출시 2~4주 측정, `타겟오디언스_전략.md` 4번 기준 사용)
| 지표 | 통과 기준 | 미달 시 액션 |
|---|---|---|
| 애드센스 승인 | 승인 완료 + 정책 위반 알림 없음 | v2 보류, BM 재설계(과금 라인 검토) |
| 저장 버튼 클릭률 | 결과 조회자 중 ≥ 10% | 공유 카피·이미지 디자인 수정 후 재측정 |
| 워터마크 UTM 유입 | 신규 방문 중 ≥ 5% | 바이럴 가정 하향, SEO를 메인 채널로 전환 |
| 세션당 수익 vs 원가 | 광고 수익 > AI+호스팅 원가 | 과금 라인 도입 전까지 AI 비중 축소 |

**게이트 통과 못 하면 v2(공유·경쟁 기능)에 투자하지 않는다 — 바이럴이 안 도는데 경쟁/랭킹을 더 붙이는 건 헛돈.**

## 2. v2 — "공유·경쟁 레이어" (v1 게이트 통과 시)

목표: v1에서 바이럴 신호가 확인됐다는 전제 하에, 공유·경쟁 동기를 증폭. 서버 DB 최초 도입.

### 포함
- **서버 DB 도입**(Vercel Postgres/Supabase 등 경량) — 아래 기능들의 공통 인프라
- 오늘의 랭킹 TOP5 + 이니셜 등록 + 비속어 필터 + IP 일일 등록 제한
- 궁합 모드 — **단, 1차는 연인궁합 1종만**(가장 타겟 적합도 높음). 베프/상사/짝사랑은 연인궁합 사용 지표 보고 추가.
- 결과 화면 연쇄 유도(퍼널) 전체 흐름 활성화(랭킹·궁합 CTA 포함)
- 타로 성향 테스트 퀴즈(무제한 플레이, AI 비용 0) — MBTI류 공유 포맷이라 바이럴 검증된 경우 효과 큼
- (옵션) v1에서 단순화했던 드릴다운 의례 부활 A/B 테스트

### 제외 → v3로
- 관계유형 4종 전체 + 4개 리더보드(연인 1종 검증 후 확장)
- 히스토리·스트릭 등 리텐션 장치

### v2 → v3 게이트 지표
| 지표 | 통과 기준 |
|---|---|
| 궁합(연인) 사용률 | 운세 조회자 중 일정 비율 이상 진입 |
| 랭킹 등록률 / 재방문 기여 | 랭킹 노출이 재방문·체류에 유의미 기여 |
| 세션당 페이지뷰 | v1 대비 상승(퍼널·퀴즈 효과 확인) |

## 3. v3 — "리텐션·확장 레이어" (안정화 후)

목표: 신규 유입이 아니라 **재방문·락인**을 만든다. (현재 기획의 최대 공백)

### 포함
- 관계유형 궁합 4종 전체 + 관계유형별 리더보드(연인 검증 결과 보고 확장)
- 히스토리 기록(로컬 → 추후 계정 기반): 지난 운세/번호 조회
- 연속 출석 스트릭 시각화
- 별자리/띠 ↔ 오늘의 카드 자동 매칭
- (BM) 직접 과금 가설 실험: 워터마크 없는 프리미엄 공유카드 / 심층 리딩 PDF 등 — 애드센스 단일 채널 의존 탈피
- 시즌 한정 카드 뒷면, 주간 인기 타로 TOP5(데이터 누적 후)

### 보류 유지
- 실제 로또 추첨 결과 비교 — "로또 미언급" 정책과 정면 충돌, 보류 유지(`기획안.md` 4순위와 동일)
- 친구 초대 보너스(추천 추적 로직 필요, 트래픽 누적 후)

## 5. 이 로드맵의 운영 규칙

1. **게이트는 사후 점검이 아니라 착수 조건이다.** v1 지표가 기준 미달이면 v2 코드를 한 줄도 쓰지 않는다.
2. **서버 DB는 v2의 신호탄이다.** v1은 끝까지 DB 없이 간다(운영·비용·승인 리스크 최소화).
3. **확장은 항상 "1종 검증 후 N종"이다.** 궁합도 연인 1종 → 검증 → 4종. 리더보드도 마찬가지.
4. **각 단계 종료 시 유닛 이코노믹스 1장을 갱신한다**(세션당 수익 vs 원가). 적자 구조면 다음 단계 투자 보류.
