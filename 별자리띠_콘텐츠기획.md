# 별자리·띠 콘텐츠 기획

기획안.md 3.3(별자리/띠별 운세 페이지)의 세부 기획. `/horoscope/[zodiac]`(12개) + `/saju/[animal]`(12개) 총 24개 페이지.

## 1. 목적

- 메인 인터랙티브 도구와 별개로, "양자리 오늘의 운세", "쥐띠 2026년 운세" 같은 검색 키워드로 자연 유입을 만드는 콘텐츠형 페이지.
- 한 번 작성하면 자주 안 바뀌는 **평가성/성격 콘텐츠**가 메인, 시의성 있는 부분(이번달/올해 흐름)은 최소화해서 유지보수 부담을 줄인다.
- 본문 안에서 자연스럽게 메인 도구(생년월일 입력 → 오늘의 운세/타로)로 연결되는 CTA를 넣어 인터랙티브 페이지로 트래픽을 보낸다.

## 2. 페이지 템플릿 — `/horoscope/[zodiac]` (별자리, 12개)

| 섹션 (H2/H3) | 내용 |
|---|---|
| H1 | "{별자리} 운세 · 성격 · 어울리는 상대" |
| H2 기본 정보 | 날짜 범위, 원소(불/물/땅/공기), 지배행성, 상징, 대표 키워드 3개 |
| H2 성격 특징 | 장점/단점 균형 있게, 3~5문단 |
| H2 애정운 스타일 | 연애할 때의 성향, 좋아하는 상대 유형 |
| H2 직장·재물운 스타일 | 일하는 방식, 돈 다루는 습관 |
| H2 잘 맞는 별자리 TOP3 | 궁합 좋은 별자리 + 짧은 이유, 각 별자리 페이지로 내부링크 |
| H2 오늘의 운세 보러가기 (CTA) | "내 생년월일로 정확한 오늘의 운세 보기" → 메인 페이지 링크 |
| 분량 | 600~800단어 목표 (최소 400단어 룰 여유 있게 충족) |

## 4. 콘텐츠 갱신 정책

- 성격/궁합/기본정보 섹션: **고정 콘텐츠**, 사이트 오픈 시 1회 작성 후 거의 갱신 안 함.
- "올해 흐름" 같은 시의성 섹션은 넣지 않거나, 넣더라도 연 1회만 갱신(예: 매년 1월 일괄 업데이트)해서 유지보수 부담 최소화.
- 메인 도구(생년월일 기반 오늘의 운세)가 "오늘" 시의성을 담당하므로, 이 24개 페이지는 시의성보다 **검색 유입 + 신뢰감 + 내부링크 허브** 역할에 집중.

## 5. 작성 방식

- 24개 페이지는 **AI로 초안 일괄 생성 → 사람이 검수/수정** 방식 추천. 매일 호출되는 운세 텍스트와 달리 1회성 정적 콘텐츠라 비용 부담이 거의 없고(전체 24페이지 합쳐도 API 비용 수백 원 수준), 표절/저품질 콘텐츠로 인한 애드센스 리스크를 검수로 방지할 수 있음.
- 생성 시 `별자리띠_기초데이터.md`의 데이터(날짜/원소/오행/궁합 등)를 프롬프트에 고정 입력으로 넣어 사실 오류를 줄이고, 섹션 구조(`별자리띠_콘텐츠기획.md`의 템플릿)를 그대로 지키도록 지시.
- 24페이지 전부 비슷한 구조라 "단순 나열/획일적" 인상을 줄 수 있으므로, 검수 단계에서 별자리·띠별로 예시·비유를 다르게 다듬어 차별화.

## 6. 내부링크 전략

- 별자리 페이지 ↔ 띠 페이지 ↔ 타로카드 페이지(`/tarot/[card-name]`) 상호 링크로 사이트 체류시간/페이지뷰 증가 → 애드센스 노출 기회 증가.
- 메인 페이지에서 생년월일 입력 시, 결과 화면에 해당 별자리/띠 페이지 링크 자동 노출(예: "양자리이신 당신, 양자리 상세 운세 더보기").
