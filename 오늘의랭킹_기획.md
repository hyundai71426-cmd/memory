# 오늘의 랭킹 (최고 점수자 TOP 5)

기획안.md 4번(핵심 로직)에 추가되는 기능의 세부 설계.

## 1. 컨셉

오늘 사이트를 방문한 사람들 중 **종합운 점수**가 가장 높은 5명을 보여주는 실시간 랭킹. 1위~5위에 든 사람은 영문 3자 이니셜을 남길 수 있다(예: "KSM"). 매일 자정 초기화.

## 2. 기존 기능과의 결정적 차이 — 서버/DB 필요

지금까지의 모든 기능(운세 점수, 타로, 행운 넘버, 오늘의 카드, 궁합 모드)은 **생년월일+날짜를 시드로 한 결정론적 계산 + 로컬 저장소 캐시**만으로 동작했다. 즉 서버에 데이터를 모아둘 필요가 없었다.

랭킹 기능은 다르다. "오늘 전체 방문자 중 상위 5명"을 매기려면 **모든 방문자의 점수를 한곳(서버 DB)에 모아 비교**해야 한다. 따라서 이 기능은 사이트에 처음으로 백엔드/DB 저장이 필요한 기능이 된다(기획안 6번 기술 스택에 반영 필요).

- 필요한 저장 데이터: 오늘 날짜, 점수, 이니셜(랭킹 진입 시), 타임스탬프, IP(또는 IP 해시, 일일 등록 횟수 제한용). 생년월일/이름 등 개인정보는 저장하지 않음.
- 매일 자정(KST) 기준 테이블 초기화 또는 날짜 컬럼으로 구분해 "오늘" 데이터만 조회.

## 3. 동작 흐름

1. 사용자가 종합운 점수를 받으면, 그 점수가 "오늘의 TOP 5" 진입 가능 점수인지 서버에 조회.
2. 진입 가능하면 "랭킹 등록하기" 버튼 노출 → 클릭 시 영문 3자 이니셜 입력 모달.
3. 이니셜 제출 → 서버에 점수+이니셜 저장 → 랭킹에 즉시 반영.
4. 메인 페이지(또는 결과 영역 하단)에 "오늘의 행운 TOP 5" 리스트 상시 노출: 순위, 이니셜, 점수.
5. 랭킹에 들지 못한 사용자도 리스트는 볼 수 있음(등록 버튼만 안 보임) — 경쟁심 자극 요소.

## 4. 이니셜 입력 규칙

- 영문 알파벳 A~Z만 허용, 정확히 3자(숫자/특수문자/한글 불가).
- **비속어 필터 필수**: 3글자 조합 수가 적어(26^3 = 17,576개) 욕설·혐오 표현을 블록리스트로 관리하기 쉬움. 영문 비속어 약어 목록(예: ASS, FUK 등 변형 포함)을 사전에 차단.
- 필터에 걸리면 "다른 이니셜을 입력해주세요" 안내 후 재입력.

## 5. 동점자 처리

- 점수가 같으면 **먼저 등록한 사람이 상위**(타임스탬프 기준 선착순).
- 6위 이하로 밀려난 기존 등록자는 랭킹에서 자동 제외.

## 6. 노출 위치 및 UI

- 메인 페이지 결과 영역 하단, 향후 확장 1순위의 "점수 옆 희소성 문구"와 같은 영역에 배치하면 시너지(예: "상위 3% 행운 + 오늘의 TOP 5 도전 가능!").
- Top 5 리스트 형태: `1위 KSM 98점 / 2위 ABC 95점 / ...`

## 7. 악용 방지 (어뷰징 고려사항)

- 같은 사람이 새로고침/재시도로 점수를 여러 번 받아 랭킹을 도배하는 것을 막아야 함 — 기존 "하루 1회 고정" 정책(4.3)이 자연히 방어막 역할을 함(같은 생년월일로는 하루에 한 점수만 나오므로 무한 재시도 불가).
- 다만 여러 가짜 생년월일을 입력해 점수를 여러 번 뽑아 랭킹을 도배하는 것은 "하루 1회 고정"만으로는 막을 수 없음. 이미 이 기능부터 서버 DB가 도입되는 시점이라 구현 비용이 거의 없으므로, **1차 출시 버전부터 IP 기준 일일 등록 횟수 제한(예: IP당 하루 3회)을 적용**하기로 확정. 완벽한 차단은 아니지만(IP 우회 가능) 진입장벽을 충분히 높여 출시 직후 랭킹이 무의미한 값으로 도배되는 상황을 방지.
- 저장 데이터에 IP(또는 IP 해시)와 등록 횟수 카운트 추가 필요 — 2번 "필요한 저장 데이터" 항목에 반영.

## 8. 기획안 반영 필요 사항

- 4번 핵심 로직에 4.7(오늘의 랭킹) 서브섹션 추가.
- 6번 기술 스택에 "랭킹 기능은 서버 DB 필요(기존 기능은 정적/로컬 처리만으로 충분했음)" 명시.
- 4.7에 "IP당 일일 등록 횟수 제한(어뷰징 방지)을 1차 출시부터 적용" 명시 필요.
