# 운세 · 타로 · 행운 넘버 사이트 기획안

## 1. 개요

생년월일 기반 오늘의 운세(종합 + 카테고리별)와 타로 3카드 스프레드를 제공하고, 금전운이 일정 점수 이상일 때만 "행운 넘버" 뽑기 버튼이 열리는 인터랙티브 사이트. 결과는 카드 이미지로 저장/공유 가능. 애드센스 수익화를 목표로 하므로 인터랙티브 기능 외에 콘텐츠형 정적 페이지를 함께 갖춘 멀티페이지 구조로 설계한다.

**타겟 오디언스**: 20~30대 여성, SNS(특히 인스타그램) 공유에 민감한 사용자층. 모든 기능/디자인/카피 결정은 이 타겟 기준으로 우선 검토한다. 상세는 `타겟오디언스_전략.md` 참고.

## 2. 사이트맵

```
/                         메인 (오늘의 운세 + 타로 + 행운 넘버 인터랙티브)
/tarot/                   타로카드 목록 페이지
/tarot/[card-name]        타로카드 78장 개별 의미 설명 페이지
/horoscope/[zodiac]       별자리별 운세 설명 페이지 (12개)
/saju/[animal]            띠별 운세 설명 페이지 (12개, 선택사항)
/guide/how-it-works       "운세는 어떻게 계산되나요" 설명 글
/guide/tarot-basics       "타로카드란 무엇인가" 입문 글
/compatibility            궁합 모드 (연인/베프/상사/짝사랑 유형 선택 → 두 사람 생년월일 → 오늘의 궁합 점수 + 케미 카드 + 관계유형별 랭킹)
/quiz/tarot-type          나의 타로 성향 테스트 (날짜 잠금 없는 무제한 플레이 사이드 콘텐츠)
/about                    사이트 소개
/privacy                  개인정보처리방침 (애드센스 필수)
```

메인 페이지는 도구(인터랙티브) 역할, 그 외 페이지는 검색 유입과 광고 노출을 위한 콘텐츠 역할로 명확히 분리한다.

## 3. 페이지별 상세

### 3.1 메인 페이지 (`/`)

- 생년월일(필수) + 이름(선택) 입력 폼
- **(변경, 2026-06-17) "오늘의 카드" 1장 즉시 공개** — 점수 계산과 무관한 워밍업 단계. 클릭 1회로 78장 중 1장이 바로 뒤집힘(정/역방향 50% 랜덤). 본 운세·타로보다 먼저 보여줘 분위기를 잡는 역할. 상세는 `오늘의카드_궁합모드_기획.md` 1번 참고.
- "타로카드 뽑기" 버튼 → 과거-현재-미래 3장 공개 (카드 뒤집기 애니메이션)
- **(변경) 점수는 한 번에 공개** — "오늘의 운세 보기"와 "타로 뽑기"를 별도 단계로 점수를 두 번 보여주지 않고, 타로 3장까지 다 뽑힌 뒤 종합운 + 카테고리별(금전/애정/직장/건강) **최종** 점수와 해설을 한 화면에서 한 번만 표시(베이스 점수는 내부 계산에만 사용, 별도로 먼저 노출하지 않음). 카테고리별 점수에는 **아이콘 표시 필요**(종합/금전/애정/직장/건강 5종, 이미지 에셋 결정 2026-06-17 — 텍스트만으로는 가독성 부족, v1 제작 대상에 포함).
- "행운 넘버 뽑기" 버튼은 게이트 없이 항상 활성화. 금전운 점수가 기준(예: 70점) 이상이면 결과에 특별 표시(골드 효과/뱃지 등)만 추가(상세는 `행운넘버_매핑규칙.md` 참고)
- 행운 넘버 6개 공개 (일부는 타로/생년월일 숫자 반영). **"로또"라는 단어나 번호 출처 설명은 노출하지 않음** — 1~45 범위·6개·오름차순 표시 형식만으로 사용자가 의미를 유추하도록 함(애드센스 도박 콘텐츠 분류 회피 목적, 상세는 `행운넘버_매핑규칙.md` 참고)
- 행운 넘버는 메인 후크가 아닌 보너스 콘텐츠로 취급. 메인 공유 후크는 "오늘의 행운 컬러"(쿠팡파트너스 검색 키워드 딥링크 연결, 상세는 `행운컬러_큐레이션_기획.md` 예정)
- "결과 카드 저장하기" 버튼 → 운세+타로+행운 넘버를 합친 이미지 다운로드. **인스타 스토리 비율(9:16) 기본 제공 + 정사각형(피드용) 옵션**, 작은 워터마크(사이트명/URL) 포함(`타겟오디언스_전략.md` 참고)
- 광고 배치: 입력 폼과 결과 영역 사이, 결과 영역 하단 (클릭 유도 위치 금지)

### 3.2 타로카드 페이지 (`/tarot/[card-name]`)

78장 각각에 대해 카드 이미지, 정방향/역방향 의미, 연애·직업·금전 관점 해석, 관련 카드 추천(내부 링크)을 담은 정보성 글(400단어 이상). 메인에서 뽑힌 카드로 자연스럽게 링크 연결.

`/tarot/` 목록 페이지는 단순 리스트가 아닌 **카드 뒤집기 갤러리**로 구현 — 78장이 카드 뒷면으로 깔려 있고 클릭하면 그 자리에서 앞면+짧은 의미가 공개됨(점수/행운 넘버와 무관, 언제든 몇 장이든 탐색 가능). 메이저/마이너 슈트별 필터 제공. 체류시간 증대 목적의 상세 설계는 `체류시간_증대_기획.md` 참고.

### 3.3 별자리/띠별 운세 페이지

각 별자리·띠의 성격, 애정·직장운 스타일, 어울리는 상대 등을 다루는 정보성 글(600~800단어). 시의성 섹션 없이 고정 콘텐츠로 운영해 유지보수 부담을 줄이고, 검색 유입 + 메인 도구로의 내부링크 허브 역할에 집중. 세부 템플릿은 `별자리띠_콘텐츠기획.md`, 기초 데이터는 `별자리띠_기초데이터.md` 참고.

### 3.4 가이드 페이지

운세 계산 원리(알고리즘을 일반인이 이해할 수 있게 풀어서 설명), 타로카드 역사와 기본 개념 등 신뢰도를 높이는 콘텐츠.

### 3.5 필수 정책 페이지

`/privacy`(개인정보처리방침), `/about` — 애드센스 승인 심사에 필요.

## 4. 핵심 로직

### 4.1 타로 드릴다운 뽑기 (사용자 인터랙션)

1. 78장을 39장씩 2덱으로 face-down 표시 → 사용자가 1덱 클릭 선택 (39장 남음)
2. 남은 39장을 13장씩 3그룹(과거/현재/미래)으로 동시에 face-down 표시
3. 각 그룹에서 사용자가 직접 1장씩 클릭해 뒤집기 → 과거/현재/미래 카드 확정 (총 클릭 4회: 덱 선택 1회 + 카드 선택 3회)
4. 정방향/역방향은 카드를 실제로 뒤집을 때 50% 랜덤 결정

### 4.2 점수 산출

- **베이스 점수**: 생년월일 + 오늘 날짜를 시드로 한 결정론적 랜덤 → 같은 사람이 같은 날 다시 봐도 동일. 카테고리별(종합/금전/애정/직장/건강) 0~100점.
- **타로 가중치**: 카드 78장마다 카테고리별 가중치 태그를 미리 정의(예: 금전 +10, 애정 -5 등), 역방향이면 가중치 부호 반전. 전체 78장 값은 `타로카드_가중치.md` 참고.
- **최종 점수** = 베이스 점수 + (과거+현재+미래 카드 가중치 합), 카테고리별로 동일하게 계산 후 0~100 범위로 클램핑.
- 결과 화면(타로 3장까지 다 뽑힌 뒤, 한 번만 공개되는 화면)에 "베이스 65점 + 타로 영향 +9 = 최종 74점" 식으로 산출 근거를 보여줘 신뢰감을 줌. 베이스 점수만 따로 미리 보여주지 않음(3.1 참고).
- 점수를 실제 해설 텍스트로 변환하는 방식(AI+템플릿 하이브리드, 비용/안전장치 포함)은 `운세텍스트_생성방식.md` 참고.

### 4.3 하루 1회 고정 (재도전 방지)

- 타로 뽑기는 하루 1회만 허용. 생년월일+오늘 날짜를 키로 로컬 저장소에 결과(선택한 덱, 뽑힌 3장, 최종 점수)를 저장.
- 같은 날 재방문 시 드릴다운 인터랙션을 다시 띄우지 않고 저장된 결과를 그대로 표시(슬롯머신처럼 "원하는 결과 나올 때까지 재시도"하는 패턴 방지).

### 4.4 행운 넘버 생성

- **의미 있는 번호 3개**(타로 3장 → 숫자 변환, 중복 시 생년월일로 대체) + **결정론적 랜덤 번호 3개**(생년월일+오늘 날짜 시드) = 총 6개, 1~45 범위 내 중복 없이 구성. 결과 화면에는 숫자만 노출하고 어떤 카드/생년월일에서 왔는지는 설명하지 않음(힌트 없음 원칙). 변환 규칙 상세는 `행운넘버_매핑규칙.md` 참고.
- **게이트 없음(변경, 2026-06-17)**: 활성화 조건을 두지 않고 누구에게나 항상 6개를 공개한다. 최종 금전운 점수 ≥ 70점이면 결과에 특별 표시(골드 효과/뱃지 등)만 추가해 희소성 느낌을 살린다. 기존 "70점 미달 시 비활성" 방식은 재도전 불가(하루 1회 고정, 4.3) 정책과 결합되면 "오늘 막혔다"는 막다른 길 경험이 돼 폐기.

### 4.5 오늘의 카드 (변경, 2026-06-17 — 역할 재정의)

- 3카드 드릴다운(4.1)보다 먼저, 생년월일 입력 직후 카드 1장을 가볍게 공개하는 워밍업 인터랙션(클릭 1회, 점수 계산에 영향 없음). 기존 "운명 비틀기"라는 위로 보상 용도는 폐기(4.4에서 행운 넘버 게이트 자체를 없앴으므로). 이제는 그 용도와 무관하게, 메인 플로우 진입 시 가장 먼저 보여주는 독립적인 "오늘의 카드" 한 장으로 역할을 재정의.
- 생년월일+오늘 날짜 키로 하루 1회 고정(4.3과 동일 정책). 세부 설계는 `오늘의카드_궁합모드_기획.md` 참고(위로 보상 관련 1.2의 3~4단계, 1.4는 삭제 필요 — 갱신 예정).

### 4.6 궁합 모드

- 두 사람 생년월일(정렬, 순서 무관) + 오늘 날짜를 시드로 **매일 바뀌는** 궁합 점수와 케미 카드 1장 산출.
- 베이스 점수(결정론적 랜덤) + 케미 카드의 애정 가중치(`타로카드_가중치.md`) = 최종 궁합 점수, 0~100 클램핑.
- 하루 1회 고정(생년월일 페어+날짜 키), 행운 넘버 기능과는 독립적. 세부 설계는 `오늘의카드_궁합모드_기획.md` 참고.
- **관계 유형 확장**: 연인궁합/베프궁합/상사궁합/짝사랑궁합 4종 선택 후 진행. 유형마다 사용하는 가중치 컬럼(애정/직장)과 해석 톤이 다름. 각 유형별로 별도의 "오늘의 궁합 랭킹 TOP5"도 운영(4.7과 동일한 DB 구조에 `relation_type` 컬럼만 추가해 재사용). 세부 설계는 `관계궁합_확장_기획.md` 참고.

### 4.7 오늘의 랭킹 (TOP 5)

- 오늘 방문자 중 종합운 점수 상위 5명을 보여주는 실시간 랭킹. 진입한 사람은 영문 3자 이니셜을 남길 수 있음(비속어 필터 적용).
- 매일 자정(KST) 초기화. 동점자는 선착순(타임스탬프 기준) 우선.
- **어뷰징 방지**: 가짜 생년월일 반복 입력으로 랭킹을 도배하는 것을 막기 위해 **IP당 일일 등록 횟수 제한(예: 3회)을 1차 출시부터 적용**(서버 DB가 이미 필요한 기능이라 구현 비용 거의 없음). 세부는 `오늘의랭킹_기획.md` 7번 참고.
- **기존 기능과 달리 서버 DB 저장이 필요한 첫 기능**(전체 방문자 점수를 한곳에서 비교해야 하므로) — 6번 기술 스택 참고. 세부 설계는 `오늘의랭킹_기획.md` 참고.
- 같은 DB 구조는 4.6의 "관계유형별 궁합 랭킹"(연인/베프/상사/짝사랑 각각 TOP5)에도 `relation_type` 컬럼만 추가해 재사용. 세부 설계는 `관계궁합_확장_기획.md` 참고.

### 4.8 결과 화면 연쇄 유도 (퍼널)

- 하루 1회 고정(4.3) 정책상 "재시도"로 체류시간을 늘릴 수 없으므로, 각 결과 화면 하단에 "다음으로 해볼 거리" CTA를 1~2개 제시해 한 세션 안에서 여러 기능을 순서대로 타도록 유도(**오늘의 카드(워밍업)**→운세+타로(최종 점수 한 번에 공개)→**오늘의 행운 컬러(메인, 항상 제공)**→행운 넘버(보너스, 게이트 없음)→오늘의 랭킹→궁합 모드→78장 갤러리).
- 모든 CTA는 강제 퍼널이 아닌 제안 수준이며 항상 건너뛸 수 있음. 세부 흐름표는 `체류시간_증대_기획.md` 참고.

## 5. 애드센스 정책 준수 사항

- **도박 연관 표현 회피**: "로또" 등 도박을 직접 지칭하는 단어·번호 출처 설명·구매 유도 문구를 일절 사용하지 않음(애드센스 도박/게임 콘텐츠 분류 회피 목적). 결과 화면에는 "참고용 콘텐츠이며 실제 결정의 근거로 사용하지 마세요"라는 일반 안내 문구만 고정 노출(`운세텍스트_생성방식.md` 5번과 동일 정책).
- **콘텐츠 깊이**: 타로/별자리/가이드 페이지는 각 400단어 이상, 독창적 해설 포함 (단순 나열 금지).
- **구조화**: 각 콘텐츠 페이지는 H2/H3 위계와 단락 3~5줄 기준 준수.
- **광고 배치**: 본문 사이·결과 하단 등 자연스러운 위치에 배치, 버튼 옆이나 닫기처럼 보이는 위치 금지.
- **모바일 밀도 가드레일**: 화면 1개(스크롤 전 영역)에 광고 1개 초과 노출 금지, 전체 페이지 광고 밀도 30% 이하, 최초 뷰포트 광고 비율 15% 이하, 페이지당 광고 단위 3~5개 권장. 핵심 액션 버튼/카드 클릭 영역 주변·입력 폼 근처는 배치 금지 구역. 상세는 `모바일UX_광고전략_기획.md` 참고.
- **필수 페이지**: 개인정보처리방침, 사이트 소개 페이지 사전 준비.

## 6. 기술 스택 제안

정적 페이지가 많은 멀티페이지 구조이므로 Next.js(또는 단순 정적 HTML 다수)를 권장. 메인 인터랙티브 영역은 React 컴포넌트로 구현, 결과 카드 저장은 html-to-canvas 류 라이브러리 사용. 타로/운세 데이터(78장 카드 정보, 별자리 설명 등)는 JSON으로 관리해 콘텐츠 페이지 자동 생성.

**서버 DB 필요 시점**: 운세/타로/행운 넘버/오늘의 카드/궁합 모드는 모두 생년월일+날짜 시드 기반 계산 + 로컬 저장소 캐시만으로 동작해 별도 서버 DB가 필요 없다. 단, "4.7 오늘의 랭킹"은 전체 방문자 점수를 한곳에서 비교해야 하므로 이 기능을 넣는 시점부터 서버 DB(예: Vercel Postgres, Supabase 등 경량 옵션)가 필요해진다.

**모바일 우선 원칙**: 전체 서비스는 모바일 뷰포트(360~430px) 기준으로 먼저 설계하고 데스크탑은 확장형으로 처리. 광고 슬롯은 항상 고정 높이로 선언해 레이아웃 시프트(CLS)를 방지. 상세는 `모바일UX_광고전략_기획.md` 참고.

**분석 도구**: GA4(또는 동급 무료 분석 도구) 도입을 1차 출시 범위에 포함 — SNS 공유 전략이 실제로 작동하는지 측정하기 위한 최소 요건. 상세는 `타겟오디언스_전략.md` 4번 참고.
