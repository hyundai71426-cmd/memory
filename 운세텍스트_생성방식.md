# 운세 텍스트 생성 방식

기획안.md 4.2(점수 산출) 결과를 사용자에게 보여줄 "말"로 바꾸는 부분의 세부 기획.

## 1. 텍스트가 필요한 지점

| 위치 | 내용 | 길이 |
|---|---|---|
| 종합운 해설 | 오늘의 총운 점수에 대한 한 줄 요약 + 짧은 해설 | 2~3문장 |
| 카테고리별 해설 (금전/애정/직장/건강) | 각 점수에 대한 짧은 해설 | 1~2문장 x 4 |
| 타로 3카드 해석 | 과거/현재/미래 카드를 엮은 스토리형 해설 | 3~5문장 |
| 점수 산출 근거 문구 | "베이스 65점 + 타로 영향 +9 = 최종 74점" 같은 정형 문구 | 1문장 (템플릿 고정, AI 불필요) |

핵심: **하루 1회 캐시**(기획안 4.3)이므로 같은 사람이 새로고침해도 같은 텍스트가 재사용된다. 즉 텍스트 생성은 "사람 수 x 1일 1회"만 발생 — 비용 부담이 거의 없다.

## 2. 생성 방식: AI + 템플릿 하이브리드

- **기본**: 저가 모델(Claude Haiku급, 입력 $1/출력 $5 per M토큰 수준) API 호출로 생성.
- **입력 프롬프트에 고정 포함할 정보**: 카테고리별 최종 점수, 점수 구간(낮음/보통/좋음/매우좋음), 뽑힌 타로 3장(이름+정역방향), 오늘 날짜.
- **출력 형식**: 정해진 길이(문장 수)와 톤을 프롬프트에 명시해 일관성 유지. JSON으로 `{종합, 금전, 애정, 직장, 건강, 타로해석}` 필드를 받아 그대로 화면에 매핑.
- **톤 가이드**: 친근한 존댓말, 점수가 낮아도 "주의가 필요한 날" 정도로 표현하고 단정적/공포 조성 문구 금지(예: "큰일 난다", "절대 ~하지 마세요" 같은 표현 지양). 애드센스 콘텐츠 정책 및 사용자 경험 모두에 도움.
- **타겟(2030 여성/SNS 공유 민감군) 보강**: 친근한 구어체 표현은 가볍게 허용(예: "오늘 운세 찐맛집", "이 카드 좀 소름")하되 과도한 신조어 남발은 금지 — 톤이 빨리 촌스러워지지 않게 "친근하지만 과하지 않게"가 기준. 상세는 `타겟오디언스_전략.md` 참고.

## 3. 폴백(템플릿) 전략

AI 호출 실패, 응답 지연, 트래픽 급증으로 인한 비용/속도 이슈 시 즉시 전환 가능하도록 점수 구간별 템플릿 문장 풀을 미리 준비한다.

- 점수 구간: 0~20(주의) / 21~40(보통 이하) / 41~60(보통) / 61~80(좋음) / 81~100(매우 좋음) — 카테고리별로 동일 구간 체계 사용.
- 각 구간 x 카테고리(5개)마다 문장 풀 3~5개를 미리 작성해두고 랜덤 선택(단, 하루 1회 고정 캐시 규칙은 동일 적용 — 같은 사람은 같은 날 같은 문장).
- 타로 해석도 카드별로 미리 써둔 짧은 정/역방향 한 줄 해설(`타로카드_가중치.md`에 카드별 의미 주석 추가 예정)을 조합해 폴백 문장 구성.
- **운영 정책 (확정)**: 평소엔 AI 우선 사용. **일일 API 비용 누적액이 사전 설정한 예산 상한(예: 5만 원/일, 운영 단계에서 실측 트래픽 보고 조정)을 넘으면 그날 자정까지 자동으로 템플릿 모드로 전환**한다. 건수 기준이 아닌 비용 기준으로 트리거하는 이유는, 건수와 실제 비용이 항상 비례하지 않기 때문(응답 길이 변동 등).
- **복귀 정책 (확정)**: 템플릿 모드로 전환된 날은 자정까지 그대로 유지하고, 다음날 00시(KST)에 자동으로 AI 모드로 리셋한다. 트래픽이 빠지면 즉시 복귀하는 방식이나 수동 확인 후 복귀 방식은 1차 버전에서는 채택하지 않음(운영 개입 없이 단순하게 운영).
- **모니터링**: 일일 누적 비용을 추적할 수 있는 카운터(또는 API 제공사의 사용량 대시보드)를 두고, 예산 상한 도달 시 전환 이벤트를 로그/알림으로 남겨 운영자가 사후에 확인 가능하게 한다(코딩 단계에서 구현).

## 4. 비용 가늠 (참고용, 이전 논의 정리)

- 1일 1회 캐시 기준으로 "순수 신규 사용자 수 = API 호출 수".
- 저가 모델 기준 1회 호출(점수+카드 텍스트 한 번에 묶어서 요청) 비용은 1~2원 수준으로 추정.
- 일 1만 명 방문 가정해도 일 비용 1~2만 원대 → 운영 가능한 수준. 이를 참고해 1차 일일 예산 상한을 **5만 원**으로 설정(약 일 2.5~5만 명 방문 수준에서 자동 전환되는 셈). 실제 운영 데이터가 쌓이면 조정.
- 급격한 트래픽 증가(바이럴 공유 등) 시에는 3번 폴백 전략으로 자동 전환되어 비용 상한선 확보.

## 5. 안전장치

- 출력 글자 수 상한을 두어 너무 길어지는 응답 방지(요금 폭증 방지).
- 금칙어 필터(자살/자해/극단적 표현, 특정인 비방, 의료·법률 단정 조언 등) 적용 — 운세 콘텐츠 특성상 민감 표현이 섞일 가능성을 줄임.
- 모든 결과 화면에 "참고용 콘텐츠이며 실제 결정의 근거로 사용하지 마세요" 안내 문구 고정 노출(애드센스 도박 오해 방지 문구와 함께 배치 가능).
