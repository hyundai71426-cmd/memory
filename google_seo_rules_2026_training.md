# 구글 SEO 규칙 (2025~2026 최신)

---

## 1. 알고리즘 구조 변화 요약

- **Helpful Content System** — 2024년 3월 코어 랭킹 시스템에 완전 통합. 더 이상 별도 알고리즘이 아님
- **Quality Rater Guidelines** — 2025년 9월 11일 최신판(182페이지) 적용 중. YMYL 정의 확장 및 AI Overview 평가 예시 추가
- **Page Experience** — 2023년 독립 신호 폐지. Core Web Vitals는 전체 품질 평가 신호로 계속 반영
- **코어 업데이트 주기** — 2025년 3회(3월·6월·12월), 2026년 3월 역대 최대 변동성 기록. 5월 추가 업데이트 진행

---

## 2. Core Web Vitals 기준 (2026)

| 지표 | 기준값 | 설명 |
|---|---|---|
| LCP | ≤ 2.5초 | 최대 콘텐츠 페인트 (로딩 속도) |
| INP | ≤ 200ms | 상호작용 응답 속도 (FID 대체, 2024.3~) |
| CLS | ≤ 0.1 | 누적 레이아웃 이동 (시각 안정성) |

> LCP 3초 초과 페이지는 유사 품질 콘텐츠 대비 트래픽 23% 더 손실

---

## 3. E-E-A-T (핵심 평가 기준)

| 축 | 의미 |
|---|---|
| **Experience** | 주제에 대한 직접 경험 (현장·실사용·실거래) |
| **Expertise** | 자격·배경·콘텐츠 깊이를 통한 전문성 |
| **Authoritativeness** | 다른 전문가·사용자에게 참고 출처로 인정받는 권위 |
| **Trustworthiness** | 정확성·투명성·보안·전반적 평판 |

- 저자 정보(이름, 자격, 약력)를 페이지에 명시
- 전문 분야를 좁게 유지하고 해당 주제에 집중
- 외부 매체 기고, 업계 행사 발표 등 외부 신뢰도 축적

---

## 4. 콘텐츠 품질 원칙

### ① 사람 중심 콘텐츠 (People-First)
- 콘텐츠가 검색엔진이 아닌 실제 사람을 위해 만들어졌는지가 핵심 판단 기준
- 사용자가 읽고 나서 "찾던 게 이거야"라고 느끼는 콘텐츠
- 검색 순위를 위해 작성된 콘텐츠는 코어 업데이트로 점점 더 걸러짐

### ② Information Gain (정보 독창성)
- 경쟁 페이지 대비 해당 페이지가 제공하는 고유한 가치를 수치로 평가
- 신선하고 독창적인 정보 = 상위 노출 핵심 요소
- 이미 웹에 넘쳐나는 정보를 재가공하는 것은 불리

### ③ 원본 데이터·1차 경험 우선
- 직접 리서치, 자체 데이터, 직접 테스트, 실제 사례 기반 분석 우대
- 자체 보유 데이터(고객 리서치, 내부 분석, 플랫폼 인사이트)를 콘텐츠 자산으로 활용
- 접근이나 전문성이 필요한 콘텐츠일수록 유리

### ④ 콘텐츠 최신화
- 기존 게시 콘텐츠를 정기적으로 점검하고 업데이트
- 더 이상 관련 없는 콘텐츠는 삭제보다 개선 우선
- 날짜만 변경하는 것은 효과 없음 (구글은 실제 수정 시점을 감지)

### ⑤ 검색 의도 충족
- 키워드 포함 여부가 아니라 실제 검색 의도를 충족하는지 평가
- 같은 키워드도 사용자 수준에 따라 다른 표현으로 검색됨 → 독자 층위를 고려해 작성

---

## 5. AI 콘텐츠 정책

- AI 콘텐츠 자체는 금지 아님. "AI가 만들었는지 사람이 만들었는지는 상관없다. 유용한지가 기준" (Google John Mueller, 2025.11)
- **허용**: AI를 도구로 활용하되, 전문성 있는 사람이 편집·검수·관점 추가
- **불이익**: 사람 검토 없이 대량 발행된 AI 생성 글, 사실 확인 없는 AI 출력물, 얕고 반복적인 AI 패턴

---

## 6. 기술적 SEO 체크포인트

### 타이틀 태그
- 2026년 기준 구글이 약 76%의 타이틀을 자체 변경 중
- 페이지 내용과 일치하는 타이틀이 최우선. 동떨어진 타이틀은 구글이 임의 교체
- 핵심 키워드를 앞쪽에 배치, 키워드 반복·도배 금지

### 구조화 데이터
- Schema.org 마크업으로 콘텐츠 유형(기사, 제품, FAQ, 리뷰 등) 명시
- AI Overview·지식 패널 등 리치 결과 노출 가능성 향상

### 사이트 구조
- 명확한 URL 구조, 내부 링크 체계 유지
- robots.txt, sitemap.xml 최신 상태 유지
- Google Search Console 정기 점검 (색인 오류, CTR, 평균 순위)

### 모바일 최적화
- 모바일 퍼스트 인덱싱 기준으로 평가
- 모바일 페이지 속도와 UX가 데스크톱보다 중요

---

## 7. 백링크

- 코어 업데이트는 온페이지 콘텐츠 품질에 집중. 백링크만으로 순위 하락을 되돌릴 수 없음
- 백링크는 여전히 유효하나, 콘텐츠 품질이 선행 조건
- 자연 발생 백링크(언급·인용)가 인위적 링크빌딩보다 안전하고 효과적

---

## 8. 절대 금지 사항

| 금지 항목 | 이유 |
|---|---|
| 검색엔진을 위한 키워드 도배 | 알고리즘 조작으로 간주, 품질 페널티 |
| 사람 검토 없는 AI 대량 발행 | E-E-A-T 기준 미달, 스팸으로 분류 |
| 날짜만 바꿔 재발행 | 구글이 실제 수정 시점 감지 |
| 얇은 콘텐츠(Thin Content) | Helpful Content 기준 미달 |
| 조작적 링크 스킴 | 스팸 업데이트 대상 |
| 사실 오류·출처 불명 정보 | Trustworthiness 하락 |

---

## 9. 체크리스트 (발행 전)

- [ ] 검색 의도를 실제로 충족하는 콘텐츠인가
- [ ] 경쟁 페이지 대비 독창적인 정보·관점이 있는가
- [ ] 저자 정보(이름·전문성)가 명시되어 있는가
- [ ] 1차 경험 또는 자체 데이터가 포함되어 있는가
- [ ] LCP ≤ 2.5초, INP ≤ 200ms, CLS ≤ 0.1 충족
- [ ] 타이틀이 페이지 내용과 일치하는가
- [ ] 모바일에서 정상 표시되는가
- [ ] 구조화 데이터(Schema) 적용되어 있는가
- [ ] 내부 링크 체계가 잡혀 있는가
- [ ] Google Search Console에서 색인 오류 없는가

---
