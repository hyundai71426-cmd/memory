# 모바일 UX · 광고 전략 기획

전체 서비스를 모바일 기준으로 설계하고, "사용자 경험 방해 최소화"와 "광고 수익 극대화"를 동시에 달성하기 위한 원칙. 구글 공식 가이드(애드센스 광고 배치/뷰어빌리티 가이드, Better Ads Standards) 기준을 이 사이트의 인터랙션 구조에 맞게 적용했다.

## 1. 모바일 우선 설계 원칙

- 모든 화면은 모바일 뷰포트(360~430px 폭) 기준으로 먼저 디자인하고, 데스크탑은 확장형으로 처리.
- 터치 타겟(카드, 버튼)은 최소 44x44px 이상 확보 — 카드 뒤집기 드릴다운(4.1)처럼 연속 클릭이 필요한 인터랙션에서 오탭 방지가 특히 중요.
- 세로 스크롤 중심 레이아웃. 가로 스크롤/가로 배치 카드 그리드는 지양(78장 갤러리도 세로 그리드로).
- 결과 화면 전환 시 레이아웃 흔들림(CLS) 방지를 위해 광고 영역은 항상 고정 높이로 미리 자리를 잡아둔다(광고 로딩 전후로 본문이 밀리지 않게).
- 결과 카드 "저장/공유 이미지"에는 광고를 절대 포함하지 않음 — 공유되는 이미지 자체는 깨끗하게 유지.

## 2. 광고 배치 — 위치별 가이드

| 위치 | 형태 | 비고 |
|---|---|---|
| 화면 최상단(above the fold) | 320x100 배너 1개 | 모든 모바일 페이지 공통, 입력 폼/카드 인터랙션과 최소 1뷰포트 거리 확보 |
| 결과 영역 사이(운세 점수 → 타로 결과 사이 등) | 인피드형 1개 | "다음 결과 보려면 스크롤"하는 자연스러운 끊김 지점에 배치 — 4.8 연쇄 유도 흐름과 겹치지 않게 CTA 버튼과는 충분히 띄움 |
| 결과 영역 최하단 | 인피드형 1개 | 결과를 다 본 후 자연스러운 위치 |
| 타로/별자리/띠 콘텐츠 본문 | "문단 2개 후", "본문 종료 후" 각 1개 | 글 읽는 흐름을 끊지 않는 지점 — 구글이 가장 권장하는 인게이지먼트 포지션 |
| 화면 하단 고정(앵커 광고) | 앵커 배너 1개 | 모바일에서 성능이 가장 좋은 포맷. 닫기(X) 버튼 필수 제공, 결과 카드 저장/공유 버튼과 겹치지 않는 위치로 조정 |

**절대 배치 금지 구역**: 카드 뒤집기 클릭 영역 주변, "오늘의 운세 보기"/"타로카드 뽑기"/"운명 비틀기" 등 핵심 액션 버튼의 상하 좁은 여백, 입력 폼 내부/근처. 오인클릭 유발 위치는 애드센스 정책 위반이자 신뢰도 손상 요인.

## 3. 밀도 가드레일 (구글 권고 기준)

- **모바일 1개 화면(스크롤 없이 보이는 영역)에 광고 1개 초과 노출 금지** — 여러 광고가 동시에 보이지 않게.
- **전체 페이지 광고 밀도 30% 이하** (광고 높이 합 ÷ 본문 전체 높이) — Better Ads Standards 모바일 기준.
- **최초 뷰포트(스크롤 전 화면)에서 광고가 차지하는 비율 15% 이하** — 2025년 이후 강화된 정책 기준, 위반 시 페이지 단위로 패널티 가능.
- 페이지당 광고 단위 총 3~5개 권장(메인 인터랙티브 페이지처럼 콘텐츠가 짧고 인터랙션이 많은 페이지는 적게, 타로/별자리 같은 긴 콘텐츠 페이지는 상한선 안에서 조금 더).
- 광고 개수를 늘리는 것보다 **노출 기회(페이지뷰·체류시간) 자체를 늘리는 것**이 수익에 더 효과적 — 이미 설계한 4.8 연쇄 유도, 78장 갤러리, 타로 성향 퀴즈(`체류시간_증대_기획.md`)가 광고 수익 전략과도 직결됨.

## 4. Auto Ads + 수동 배치 하이브리드

- 메인 인터랙티브 페이지: **수동 배치**로 위치를 엄격히 통제(인터랙션 방해 금지가 최우선이므로 자동 배치에 맡기지 않음).
- 타로/별자리/띠 콘텐츠 페이지(정적 페이지 다수): **Auto ads 활성화**해 위치 최적화를 구글에 위임, 운영 부담을 줄임. 단, 밀도 상한 설정값(위 3번 기준)을 애드센스 콘솔에서 직접 제한.
- 앵커 광고·전면(vignette) 광고는 애드센스 콘솔의 "전체화면 광고" 설정에서 개별 on/off — 전면 광고는 페이지 전환이 거의 없는 이 사이트 구조상 사용자 흐름을 크게 끊을 수 있어 1차 버전에서는 비활성 권장.

## 5. 실제 적용 체크리스트 (코딩 단계 참고용)

- [ ] 광고 슬롯은 항상 고정 높이 컨테이너로 선언(CLS 방지)
- [ ] 메인 페이지 above-the-fold 광고와 "오늘의 운세 보기" 버튼 사이 최소 간격 확보
- [ ] 결과 카드 저장 이미지 렌더링 시 광고 DOM 제외 처리
- [ ] 실제 모바일 기기(개발자 도구 아님)에서 광고 노출 후 레이아웃/탭 영역 검증
- [ ] 애드센스 콘솔에서 모바일 밀도 상한 설정 확인

## 6. 기획안 반영 필요 사항

- 5번(애드센스 정책 준수)에 모바일 밀도 가드레일과 배치 금지 구역 명시.
- 6번(기술 스택)에 "모바일 우선 반응형 설계, 광고 슬롯 고정 높이" 원칙 추가.
